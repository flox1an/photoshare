  <behavior>
    Hook signature: useAlbumViewer({ naddr }: { naddr: string })

    Returns:
    - status: 'loading' | 'ready' | 'error'
    - error: string | null
    - manifest: AlbumManifest | null
    - thumbUrls: Record<string, string>  // key = photo.thumbHash, value = object URL
    - fullUrls: Record<string, string>   // key = photo.hash, value = object URL (loaded on demand)
    - albumKey: CryptoKey | null
    - blossomServer: string
    - downloadAll(): Promise<void>
    - downloadProgress: { current: number; total: number } | null
    - loadThumbnail(index: number): void  // called by ThumbnailGrid IntersectionObserver
    - loadFullImage(index: number): void  // called by Lightbox on open

    State machine (in useEffect on mount):
    1. Read key from window.location.hash.slice(1) — CRITICAL: must be inside useEffect, never render body
    2. Call importKeyFromBase64url(fragment) → albumKey
    3. Call loadAlbumEvent(naddr) → event
    4. Call decryptManifest(event, albumKey) → manifest
    5. Set status = 'ready'
    6. On any error: set status = 'error', error = err.message

    loadThumbnail(index):
    - If thumbUrls[photo.thumbHash] already exists, skip
    - fetchBlob(blossomServer, photo.thumbHash) → ciphertext
    - base64urlToUint8Array(photo.thumbIv) → iv
    - decryptBlob(ciphertext, albumKey!, iv) → plaintext
    - URL.createObjectURL(new Blob([plaintext], { type: 'image/webp' })) → objectUrl
    - setThumbUrls(prev => ({ ...prev, [photo.thumbHash]: objectUrl }))

    loadFullImage(index):
    - Same pattern but uses photo.hash and photo.iv instead of thumbHash/thumbIv
    - Sets fullUrls[photo.hash]

    downloadAll():
    - Set downloadProgress = { current: 0, total: manifest.photos.length }
    - For each photo in sequence: fetchBlob → decryptBlob → zip.file(photo.filename, plaintext)
    - Update downloadProgress.current after each photo
    - generateAsync({ type: 'blob' }) → trigger <a download> without FileSaver.js
    - Reset downloadProgress = null after completion

    Cleanup (useEffect return):
    - Revoke ALL object URLs in thumbUrls and fullUrls on unmount
    - Use useRef to track created URLs for reliable cleanup

    blossomServer: use DEFAULT_BLOSSOM_SERVER (no per-photo server URL in PhotoEntry — confirmed in RESEARCH.md open question resolution)

    CRITICAL anti-patterns to avoid (from RESEARCH.md):
    - NEVER access window.location.hash outside useEffect
    - NEVER create RelayPool at module scope (loadAlbumEvent handles this internally)
    - ALWAYS revoke object URLs
    - NEVER load full images on mount — only on demand (loadFullImage)
  </behavior>
