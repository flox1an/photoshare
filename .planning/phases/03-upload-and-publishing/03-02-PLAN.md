  <behavior>
    sha256Hex:
    - Takes ArrayBuffer, returns Promise<string>
    - Uses crypto.subtle.digest("SHA-256", buffer) — Web Crypto only, no library
    - Returns lowercase hex string, exactly 64 characters
    - Must be computed on ENCRYPTED bytes (ciphertext), not plaintext

    buildBlossomUploadAuth:
    - Takes signer (PrivateKeySigner) and blobHashHex (string)
    - Builds kind 24242 event via applesauce-factory build() with:
      - tags: ["t", "upload"], ["x", blobHashHex], ["expiration", String(Math.floor(Date.now()/1000) + 3600)]
      - content: "Upload blob"
    - Signs with signer.signEvent(template)
    - Returns "Nostr " + btoa(JSON.stringify(signedEvent)) — standard base64 (btoa), NOT base64url
    - CRITICAL: Do NOT use uint8ArrayToBase64url here — that would break the Authorization header

    uploadBlob:
    - Takes (serverUrl: string, ciphertext: ArrayBuffer, authHeader: string, mimeType?: string)
    - PUT to `${serverUrl}/upload` with Authorization header and Content-Type
    - If response is not ok: throw Error with status code in message
    - Parse response as BlobDescriptor
    - Compute SHA-256 of ciphertext locally, compare to descriptor.sha256
    - If hashes differ: throw Error("Blossom hash mismatch: ...")
    - Return BlobDescriptor

    validateBlossomServer:
    - Takes url: string, returns Promise<boolean>
    - HEAD request to url with AbortSignal.timeout(5000)
    - Returns true ONLY if response.ok AND response.headers.get("access-control-allow-origin") is non-null
    - Catch any error and return false
    - CORS check must specifically check the header — not just HTTP 200
  </behavior>
