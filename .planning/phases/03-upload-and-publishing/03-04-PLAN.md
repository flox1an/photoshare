  <behavior>
    startUpload(photos, settings) pipeline in order:
    1. Generate fresh album key: `generateAlbumKey()`
    2. Create ephemeral signer: `createEphemeralSigner()`
    3. For each ProcessedPhoto (concurrency: p-limit(3)):
       a. setEncrypting(photo id)
       b. encryptBlob(photo.full, key) → { ciphertext: fullCipher, iv: fullIv }
       c. encryptBlob(photo.thumb, key) → { ciphertext: thumbCipher, iv: thumbIv }
       d. sha256Hex(fullCipher) → fullHash
       e. sha256Hex(thumbCipher) → thumbHash
       f. setUploading(photo id)
       g. buildBlossomUploadAuth(signer, fullHash) → fullAuthHeader
       h. buildBlossomUploadAuth(signer, thumbHash) → thumbAuthHeader
       i. uploadBlob(settings.blossomServer, fullCipher, fullAuthHeader) → fullDescriptor
       j. uploadBlob(settings.blossomServer, thumbCipher, thumbAuthHeader) → thumbDescriptor
       k. setUploadDone(photo id, { hash: fullHash, thumbHash })
       l. Accumulate PhotoEntry: { hash: fullHash, iv: uint8ArrayToBase64url(fullIv), thumbHash, thumbIv: uint8ArrayToBase64url(thumbIv), width: photo.width, height: photo.height, filename: photo.filename }
       — On ANY error: retry up to 3 times with exponential backoff (100ms * 2^attempt)
       — After 3 failed retries: setUploadError(photo id, error.message); mark photo as failed
    4. If any photo has upload error → do NOT proceed to publish; set publishError
    5. Build AlbumManifest: { createdAt: new Date().toISOString(), photos: photoEntries }
       (title is set later in ShareCard before publish — or passed in settings; keep optional)
    6. Encrypt manifest: encryptBlob(TextEncoder().encode(JSON.stringify(manifest)).buffer, key)
       → { ciphertext: manifestCipher, iv: manifestIv }
       NOTE: manifestCipher may be large (>64KB for 200 photos) — use chunked base64 (see crypto.ts note)
    7. Encode encryptedManifestB64url: chunked base64url of manifestCipher
    8. ivB64url = uint8ArrayToBase64url(manifestIv) — 12 bytes, spread-btoa is fine
    9. dTag = crypto.randomUUID()
    10. buildAlbumEvent(signer, encryptedManifestB64url, ivB64url, dTag)
    11. pool.publish(settings.relays, event) → PublishResponse[]
    12. If ANY response has ok=false → setPublishError(failures), shareLink remains null (UPLD-08 gate)
    13. If all ok=true → exportKeyToBase64url(key) → keyB64url
    14. pubkey = getSignerPubkey(signer)
    15. naddr = encodeAlbumNaddr(dTag, pubkey, settings.relays)
    16. shareLink = `/${naddr}#${keyB64url}` (relative path; ShareCard will build full URL)
    17. navigator.clipboard.writeText(window.location.origin + shareLink) — auto-copy per user decision

    RelayPool lifecycle:
    - Create RelayPool in useRef: `poolRef = useRef<RelayPool | null>(null)`
    - Initialize in useEffect: `poolRef.current = new RelayPool()`
    - Never create RelayPool at module scope (Pitfall 2)
    - After publish completes, do NOT explicitly disconnect — let GC handle it

    Manifest base64url encoding for large buffers:
    - DO NOT use uint8ArrayToBase64url(new Uint8Array(manifestCipher)) — stack overflow for 200-photo albums
    - Inline a chunked encoder:
      ```typescript
      function arrayBufferToBase64url(buf: ArrayBuffer): string {
        const bytes = new Uint8Array(buf);
        let binary = '';
        const chunkSize = 8192;
        for (let i = 0; i < bytes.length; i += chunkSize) {
          binary += String.fromCharCode(...bytes.subarray(i, i + chunkSize));
        }
        return btoa(binary).replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');
      }
      ```

    State exposed by hook: { startUpload, shareLink, isUploading, publishError }
    isUploading: true between startUpload() call and final resolution
